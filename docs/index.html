Native macOS widget for Claude Code

让 Claude Code 的状态浮在桌面上。

Claude 在终端里忙的时候，你不用一直切回去看。这个迷你信号灯会悬浮在所有窗口和全屏 Space 之上，用红黄绿告诉你该继续等、可以回看，还是需要处理。

小到不打扰，亮到一眼懂

它不是一个需要管理的窗口，而是一个安静的桌面挂件。

原生 SwiftUI

没有 Electron、没有 Python、没有辅助脚本。安装后就是一个普通 macOS App。

真正像挂件

无 Dock 图标、无菜单栏图标、无标题栏按钮。右键退出，拖动即可换位置。

覆盖所有工作区

窗口级别高于普通 App，并加入所有 Space，包括全屏应用。

三个灯号

状态足够少，才适合一眼扫过。

黄灯

Claude 正在思考、流式输出，或者正在调用工具。先别切回去盯着终端。

绿灯

上一轮已经干净结束。现在回到终端，通常就能直接看结果。

红灯

Claude Code 没运行、权限确认在等你、工具出错，或者进程已经退出。

安装教程

按这几步走，不需要配置 hooks，也不需要改 Claude Code 设置。

1

下载并解压

从 GitHub Releases 下载 `ClaudeTrafficLight.zip`，双击解压得到 `ClaudeTrafficLight.app`。

2

放进应用程序

把 App 拖到 `/Applications`。这样 Spotlight、启动台和登录项都能正常找到它。

3

第一次右键打开

如果 macOS 提示无法验证开发者，右键 App 选择 Open。也可以运行 `xattr -dr com.apple.quarantine /Applications/ClaudeTrafficLight.app`。

4

启动 Claude Code

在终端运行 `claude` 并提交一次 prompt。信号灯会读取 `~/.claude/projects` 下的会话日志。

工作原理

App 每秒做两件事：检查 Claude Code 进程是否存在，读取最新 transcript 的最后状态。

status mapping
no claude process          -> red
tool use / user prompt     -> yellow
clean assistant end turn   -> green
permission / hook failure  -> red

常见问题

为什么没有 Dock 图标？

这是设计目标。它以 LSUIElement 后台应用运行，更像桌面挂件，而不是普通窗口。

怎么退出？

右键点击信号灯，选择退出。如果找不到它，也可以在活动监视器里结束 ClaudeTrafficLight。

一直红灯怎么办？

先确认终端里真的运行了 Claude Code。然后提交一次 prompt，确保 `~/.claude/projects` 里有新的 JSONL 日志。

会上传数据吗？

不会。它只在本机读取 Claude Code 的日志和进程状态，没有网络请求，也没有遥测。
