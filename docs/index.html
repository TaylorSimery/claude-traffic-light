Features

Quiet on your desk.
Loud when it matters.

No menu bar clutter, no Dock icon, no notifications. Just one tiny pane of glass that tells you exactly where the agent is.

🪟

Frosted-glass widget

Native SwiftUI panel with vibrancy and a soft drop-shadow. Floats above other windows on every Space.

⚡

Live session reading

Polls ~/.claude/projects/**/*.jsonl every second. No network, no telemetry, no daemon.

🪶

Zero footprint

Runs as an LSUIElement background app. ~250 KB executable. Drag to move, right-click to quit.

🍎

Pure Apple stack

SwiftUI + AppKit. No Electron, no Python, no helper scripts. Universal binary signed for local run.

🔒

Local only

Reads files in your home directory. Nothing leaves the machine. Source is short and auditable.

🛠

Hackable

State logic lives in a single Swift file — adjust the rules, recompile in Xcode, run.

Status mapping

Three colors. One signal.

Claude Traffic Light reads the latest message in your active session and the CLI process state, then picks one light.

Yellow — Working

Claude is thinking, streaming, or executing a tool. The session log is moving.

Green — Done

The last turn ended cleanly with end_turn. Safe to come back.

Red — Needs you

A tool call is awaiting permission, the run errored, or the process exited.

Install

Two ways to get it running.

Prebuilt app

  1. Download ClaudeTrafficLight.zip from Releases.
  2. Move ClaudeTrafficLight.app to /Applications.
  3. First launch: right-click the app → Open, or run the command below to drop the quarantine flag.
# clear quarantine for the ad-hoc signed build
xattr -dr com.apple.quarantine /Applications/ClaudeTrafficLight.app

Build from source

Requires Xcode 15+ and macOS 13+.

git clone https://github.com/TaylorSimery/claude-traffic-light.git
cd claude-traffic-light
open ClaudeTrafficLight.xcodeproj   # or build from the CLI:
xcodebuild -project ClaudeTrafficLight.xcodeproj \
           -scheme ClaudeTrafficLight \
           -configuration Release \
           -derivedDataPath build clean build
